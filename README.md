# 基于大模型解释的工业异常监测与诊断系统

一个面向工业时序数据的异常监测与智能诊断原型系统。项目实现了从**数据输入、异常检测、异常区间提取、规则摘要生成，到大模型诊断解释与可视化展示**的完整闭环。

---

## 项目简介

在工业过程监测场景中，传感器数据通常具有多变量、时序性和波动性强等特点。为了快速验证“异常检测 + 大模型解释”这一思路，本项目构建了一个轻量级原型系统：

- 对多传感器时序数据进行异常检测
- 自动识别异常点与连续异常区间
- 统计各传感器异常情况
- 自动生成结构化中文异常摘要
- 接入大模型生成诊断解释与排查建议
- 通过 Streamlit 实现交互式展示

当前版本基于模拟工业时序数据进行流程验证，适合作为 AI 应用原型、课程项目或简历展示项目。

---

## 功能特点

- **多传感器异常检测**  
  基于 robust z-score 与阈值机制，对每个传感器分别进行异常识别。

- **整体异常判定**  
  支持根据多传感器结果生成整体异常标签与异常分数。

- **连续异常区间提取**  
  对连续异常点进行合并，输出异常区间起止位置、持续长度和主导异常传感器。

- **规则化异常摘要生成**  
  自动将检测结果转化为结构化中文分析文本，包括：
  - 总体检测结果
  - 各传感器异常统计
  - 连续异常区间分析
  - 初步诊断结论
  - 后续建议

- **大模型诊断解释**  
  接入智谱 GLM OpenAI-compatible API，根据异常摘要生成更完整的诊断解释与排查建议。

- **交互式 Web Demo**  
  基于 Streamlit 构建页面，支持阈值调节、传感器切换、结果查看与文本下载。

## 方法说明

### 1. 异常检测方法
项目使用的是 **robust z-score**，而不是普通 z-score。

对于某一传感器序列 \(x\)，定义：

\[
z_{robust} = 0.6745 \cdot \frac{x - \text{median}(x)}{\text{MAD}}
\]

其中：

\[
\text{MAD} = \text{median}(|x - \text{median}(x)|)
\]

相比均值和标准差，使用中位数与 MAD 对异常值更稳健，更适合工业异常检测场景。

### 2. 阈值机制
对每个传感器，在每个时刻判断：

\[
|z_{robust}| > T
\]

若成立，则认为该传感器在该时刻异常。  
其中 \(T\) 为异常阈值，可在页面中手动调节。

### 3. 整体异常判定
若某一时刻任意一个传感器满足越阈值条件，则该时刻被标记为整体异常时刻。

### 4. 异常区间提取
对连续异常点进行合并，形成异常区间，用于后续分析与大模型解释。
